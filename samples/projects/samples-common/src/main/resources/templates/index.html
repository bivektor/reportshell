Available Sample Reports

Different sample applications expose different reports. Run other samples to explore additional configuration options.

  • Customers Report

    Customer listing report with an advanced parameter form.

    • Nested multi-valued cascading controls (regions > countries > cities)
    • Input control parameter default value evaluation
    • Format validation for date and number parameters
    View Report

  • Customer Spend Report

    Lists all customers with total spend by product charts.

    • Subreport resolution relative to a root directory (charts).
    • Demonstrates UI when the report has no input controls.
    View Report

  • Orders Report

    Order history with filters and derived totals

    • Demonstrates report data set using a custom data source.
    • Includes a customer dropdown querying from the same custom data source.
    View Report

  • Countries by Region Report

    Lists countries filtered by region.

    • Report key "Countries" is stored in the database, mapped to the compiled file at region/Countries.jasper
    • Demonstrates how a custom report store decouples the URL key from the file path
    • Demonstrates nested path report lookup (/region/Countries) in the URL
    View Report View Report

Exploring the REST API

You can use your browser's developer tools (Inspector/DevTools) to observe the REST API requests made by the viewer. Consult the REST API reference documentation to try additional APIs and parameters not exposed in this demo viewer.
